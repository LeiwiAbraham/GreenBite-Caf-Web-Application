Sprint One Work Diary – GreenBite Café Web Application
Day	Activities Completed	Issues Faced	Solutions / Reflection
Day 1 – Project Planning	Reviewed client requirements and discussed project scope with team members. Created initial project structure and repository setup on GitHub. Planned homepage and navigation design.	Some requirements were unclear during initial discussion.	Conducted team discussion and simplified project scope to focus on HTML, CSS, and JavaScript only.
Day 2 – Wireframes & UI Design	Designed low-fidelity wireframes for homepage, menu section, booking form, and navigation bar. Started responsive CSS layout design.	Difficulty maintaining consistent layout across devices.	Used Flexbox and CSS Grid to improve responsiveness and consistency.
Day 3 – Homepage Development	Developed homepage structure using HTML and CSS. Added hero section, featured items, and responsive navigation bar.	Navigation alignment issues on smaller screens.	Adjusted media queries and improved spacing using responsive breakpoints.
Day 4 – Menu Functionality	Implemented dynamic menu rendering using JavaScript arrays and DOM manipulation. Added food categories and item cards.	Initial menu structure was repetitive and difficult to maintain.	Converted static HTML menu items into reusable JavaScript-generated components.
Day 5 – Cart & Local Storage	Developed shopping cart simulation using JavaScript. Added add-to-cart buttons and local storage support.	Cart items duplicated after refreshing the page.	Fixed rendering logic by clearing previous cart elements before updating data.
Day 6 – Reservation Form & Testing	Created reservation/contact form with JavaScript validation. Performed responsive testing and debugging across multiple screen sizes.	Validation errors were inconsistent for empty inputs.	Added proper input checks and improved validation conditions.
Day 7 – Final Review & Sprint Demo	Conducted final testing, fixed UI bugs, and prepared sprint presentation. Demonstrated working prototype to client and academic staff.	Minor CSS conflicts appeared after merging team components.	Standardised CSS naming conventions and reviewed merged files carefully. Learned importance of teamwork and version control.
